> XaaS, Anything as a Service

XaaS
====

Quelques terminologies obscures et rébarbatives de cet engouement pour le .aaS :

* HaaS : Hardware As A Service
* DaaS : Desktop As A Sercice ; Database As A Service
* ITaaS : IT As A Service
* VaaS : Video As A Service
* STaaS : Storage As A Service
* PRAaaS : le Plan de Reprise d’Activité est délivré en tant que service
* RaaS : Rack As A Service, un mode de facturation de l’infrastructure

Les nuages
==========

On parle également de Cloud Public, Cloud Privé, Cloud Hybrid.

* Le Cloud public correspond à l’achat de service à un opérateur de service, selon les entreprises ceux-ci sont utilisés pour des applications peu critiques.
* Le Cloud privé correspond simplement à un « cloud interne » au sein de l’entreprise. Cette pratique ressemble plus à de la virtualisation qu’à du Cloud Computing. Certains fournisseurs vendent des packs all-in-one pour monter rapidement une architecture virtualisée dans une entreprise. Les raisons pour employer du Cloud privé sont surtout accès sur la protection des données en interne. Concerver les données en interne est un vœux que les entreprises font avec leurs clients, il ne faut pas seulement penser aux données de l’entreprise mais aussi aux données clients. Avant toute migration vers un Cloud public notamment, il est important d’avertir ses clients si ceux-ci acceptent de voir leurs informations stockées dans le nuage.
* Le Cloud hybrid représente la combinaison du Cloud public et du Cloud privé. On peut avoir une application de gestion de messagerie en mode SaaS pour le public et une architecture virtualisée en interne pour les applications propriétaires et plus critiques pour l’entreprise.
