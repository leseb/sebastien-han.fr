Dans ce billet, nous revenons plus précisément sur les SaaS. Une analyse un peu plus précise du concept, qui vient en complément de l’article sur les généralités du Cloud Computing.

<!--more-->

Le SaaS, Software As A Service, est un service d’applications en ligne orienté end-user. Ici l’opérateur de service fournit une une application et maintient toute l’infrastructure. Le client paie simplement un abonnement pour disposer d’une application.

La logique du SaaS tend à une dépossession du poste de travail. Ici la station de travail de la personne ne sert plus qu’à un retour d’affichage depuis un navigateur Web en utilisant les technologies RIA.

Avantages
=========

Voici une liste d’avantages propres au modèles SaaS, en plus de tous les avantages offerts par le Cloud Computing :

* Administration très simple
* Collaboration améliorée
* Prix des applications en ligne très attractif (ex: 40€/an/utilisateur pour Google Apps For Business)
* TCO proche de  0
* Réduction du TTM (Time To Market)
* API de migration des données
* Interface des application user-friendly
* Plus de notion de versionning
    * Update automatique
    * Suppression des problèmes de compatibilité: tous les utilisateurs ont la même version du logiciel

Inconvénients
=============

Voici une liste des inconvénients spécifiques au modèle SaaS, ils s’additionnent aux problématiques déjà évoquées dans l’article précédent :

* Former les employés sur la nouvelle application
* Une migration n’est jamais aisée

Exemples de fournisseurs
========================

* [Salesforces.com](http://www.salesforce.com)  : logiciel CRM
* [Google Apps For Business](http://www.google.com/apps/intl/fr/business/index.html) : suite bureautique de Google
* [Office365](http://office365.microsoft.com/en-US/online-services.aspx) : le version SaaS de la suite MS Office
